UX Iteration 4 — Setup Wizard & Connection Status

Focus: SetupWizard UX improvements, ConnectionStatus refinement, navigation hover states

Theme

Iteration 4 addressed the setup wizard for adding new machines and the connection status indicator. Both are critical user touchpoints — the wizard is the first experience a user has when expanding their fleet, and the connection status must communicate state without causing anxiety.

Changes Made

Connection Status Indicator

  • Less alarming when WS is offline but machines are online — now shows "WS offline" with a gentle pulse instead of a red "Disconnected" warning
  • Pulsing dot animation on reconnecting/connecting states using CSS keyframes
  • Three-state indicator: green (connected), yellow pulsing (reconnecting), dim gray (offline)
  • Tooltip on hover explains what each state means

Sidebar Navigation Polish

  • Tasks link visually distinct as primary nav item with accent border and background tint
  • Hover effects on all nav items (Sessions, Machines, History, Manual, Quick Help)
  • Active page indicated with left border accent and background highlight
  • Consistent 8px border-radius on all interactive nav elements

Status Color Utility Classes

  • Added CSS utility classes for consistent status colors across all components
  • .status-online, .status-offline, .status-pairing with matching opacity backgrounds
  • Ensures MachineCard, SessionCard, ConnectionStatus all use the same palette

Positive: The connection status is much better now — no false alarms. The sidebar nav feels responsive and alive with hover states. Good use of consistent status colors.

Suggestions for next iteration:

  • The sessions page could benefit from a "quick actions" context menu on session cards
  • Consider adding keyboard shortcuts documentation in the help panel
  • History page could use similar card-based styling for execution records
