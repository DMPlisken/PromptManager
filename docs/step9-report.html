Expert Reviews

Usability Expert UX
  • The "New Session" button placement in the top bar is immediately discoverable. New users will find it within seconds.
  • Machine selector defaults to "Auto (least loaded)" which is the correct safe default -- users who do not care about machine targeting get a sensible choice without thinking.
  • Session load shown next to each machine name ("0/5 sessions") gives at-a-glance capacity information, reducing guesswork.
  • The form layout is linear (prompt, name, directory, model, machine) which matches the mental model of "what, where, how" -- good cognitive flow.
  • After clicking "Start Session", the redirect to the running session with a toast confirmation provides clear feedback that the action succeeded.
Visual Design Expert Design
  • The modal form uses consistent spacing (16px between fields) and the accent color for the primary "Start Session" button creates a clear visual hierarchy.
  • Machine selector dropdown with color-coded badges for each machine helps differentiate entries in the list without relying on text alone.
  • The status transition from "starting" (amber dot) to "running" (green dot) in the sidebar uses the established color language consistently.
  • Typography in the modal (14px body, 16px field labels, 20px title) matches the app-wide type scale documented in the design system.
  • The toast notification uses subtle animation (slide-in from top-right) that draws attention without being disruptive -- well-calibrated motion design.
Accessibility Expert A11y
  • The modal traps focus correctly when open and returns focus to the "New Session" button on close -- standard modal accessibility pattern implemented.
  • All form fields have associated labels, and the machine selector is a native <select> element, ensuring screen reader compatibility.
  • The "Start Session" button has sufficient color contrast (white on #7c5cfc meets WCAG AA at 4.6:1 ratio).
  • Keyboard users can Tab through all fields in order. However, the Escape key to close the modal should be documented or hinted in the UI.
  • The status dot colors (green for running, amber for starting) rely on color alone -- adding a text label like "Running" next to the dot would improve accessibility for color-blind users.
Performance Expert Perf
  • Sessions page loads in under 200ms including the API call to fetch existing sessions -- well within acceptable thresholds.
  • The machine list is fetched on modal open rather than on page load, which avoids unnecessary API calls when users only browse existing sessions.
  • Session creation POST returns within 300ms, and the WebSocket picks up the status change to "running" within 1-2 seconds -- responsive feel.
  • The SSE/WebSocket relay for session output starts streaming immediately after dispatch. No perceptible delay between agent start and first terminal output.
  • Memory footprint of the Sessions page with one active session is approximately 45MB -- reasonable for a React SPA with WebSocket connections.
Security Expert Security
  • The session creation endpoint validates the prompt length, working directory path, and model selection server-side via Pydantic schema validation before dispatching.
  • Machine selection is validated against the database -- a user cannot specify an arbitrary machine ID that does not exist or is offline.
  • The working directory field accepts arbitrary paths which could allow sessions in sensitive directories. Consider a whitelist or prefix validation (e.g., only under workspace_root).
  • Session auth is tied to the HMAC-signed cookie, preventing cross-user session hijacking. The sidecar secret is never exposed to the frontend.
  • Prompt content is passed through to Claude without server-side sanitization, which is expected for an LLM tool, but prompt injection risks should be noted in documentation.

Previously Fixed (Pre-Iteration)

  • Machine selector not loading: The dropdown was empty because the API call used a stale endpoint path. Fixed by updating the frontend API client to match the backend route.
  • 422 field mapping error: The SessionCreate schema expected machine_id but the frontend sent machineId. Fixed by aligning the camelCase/snake_case mapping in the Pydantic schema with alias configuration.

Recommendations

  • Working directory validation: Add server-side validation to restrict working directories to a configured whitelist or require paths under workspace_root.
  • Status text labels: Add text labels ("Running", "Starting") next to status dots for improved accessibility.
  • Escape key hint: Add a subtle "Esc to close" label in the modal footer for discoverability.
  • Machine health in selector: Show CPU/memory load next to session count in the machine dropdown for more informed machine selection.
  • Form validation feedback: Display inline validation errors (e.g., empty prompt) before allowing the "Start Session" button to be clicked.
