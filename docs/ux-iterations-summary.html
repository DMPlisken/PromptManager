Bugs Found and Fixed

BUG-001 (CRITICAL) — Sessions page blank

Symptom: Sessions page renders completely blank with TypeError: Cannot read properties of undefined (reading 'substring')

Root cause: s.initialPrompt.substring(0, 60) crashes when initialPrompt is undefined

Fix: Null guards: (s.initialPrompt ?? "").substring(...)

File: SessionsPage.tsx

BUG-002 (MEDIUM) — Test output "(no output received)" despite messages

Symptom: Machine test passes but output area shows "(no output received)"

Root cause: Backend only extracted text from msg.content, but Claude stores responses in msg.metadata_json

Fix: Updated output extraction to also parse metadata_json in multiple formats

File: machines.py

BUG-003 (LOW) — Stale pairing records on wizard cancel

Symptom: Cancelled wizard leaves "Pending" machine records in database

Root cause: generate_pairing_code creates Machine record but cancel has no cleanup

Fix: (1) Return machine_id in PairingCodeResponse, (2) Frontend deletes on cancel, (3) Backend auto-cleans stale records older than 1 hour

Files: machine.py (schema) machines.py (router) SetupWizard.tsx

BUG-004 (MEDIUM) — MachineCard crash on undefined machine_uuid

Symptom: MachineCard crashes for machines in "pairing" status

Root cause: machine.machine_uuid.substring(0, 6) without null check

Fix: Changed to (machine.machine_uuid ?? "").substring(0, 6)

File: MachineCard.tsx

UX Changes -- Categorized

Visual Design

  • Tab bar: bottom accent border on active tab with transitions
  • Session sidebar cards: card backgrounds, status badges, relative timestamps
  • Machine cards: emoji platform badges, gradient backgrounds, hover elevation
  • Connection status: pulsing dot animation, less alarming offline state
  • Status color utility classes for consistency across components

Information Architecture

  • Machine health: disk space metric added, memory in human-readable format
  • History page: card-based records with group badges and variable pills
  • Empty states: centered with icon, description, and CTA button
  • Differentiation between "no items exist" and "no item selected"

Interaction Design

  • Sidebar nav: hover effects on all items
  • Tasks link: visually emphasized as primary navigation
  • Active page: left border accent indicator

Backend Quality

  • Stale pairing records auto-cleaned on GET /machines (1-hour grace period)
  • Actionable error messages for all session creation failure modes
  • Structured logging for session lifecycle events from remote agents
  • Error type differentiation: CircuitBreaker, Connection, Timeout, generic

Typography & Spacing

  • Heading sizes: H1 (28px), H2 (20px), H3 (16px)
  • Body text: 14px with 1.6 line-height
  • Card padding: 20px universal
  • Section gaps: 16px between cards, 32px between sections
  • Button padding: 8px 16px regular, 6px 12px compact

Architecture Overview

Multi-Machine Orchestration System

PromptFlow manages Claude Code CLI sessions across multiple computers from a single web dashboard.

Components

ComponentTechnologyPurpose
FrontendReact 18 + TypeScript + ViteWeb UI with real-time session monitoring
BackendPython 3.12 + FastAPI + SQLAlchemy 2.0REST API, WebSocket hub, session management
DatabasePostgreSQL 16Persistent storage for sessions, machines, templates
Orchestrator SidecarPython + FastAPIManages local Claude CLI subprocesses
Remote AgentNode.js + TypeScriptRuns on remote machines, connects via WebSocket

Communication Flow

  • Browser <-> Backend: REST API + WebSocket for real-time updates
  • Backend <-> Sidecar: HTTP + SSE for local session control
  • Backend <-> Remote Agents: WebSocket for cross-machine session dispatch
  • Agent <-> Claude CLI: Subprocess with stream-json stdout parsing

Recommendations for Future Improvements

High Priority

  • Markdown rendering for assistant messages — Claude's output includes markdown formatting that should be rendered properly in the terminal view
  • Code block syntax highlighting — Tool use inputs and code snippets should have syntax highlighting for readability
  • Token/cost display component — Show real-time cost tracking per session with cumulative totals

Medium Priority

  • Virtual scrolling for long sessions — Sessions with 10K+ messages will benefit from virtualized list rendering
  • Session search and filtering — Find sessions by prompt content, machine, date range, or status
  • Approval router — Dedicated REST endpoints for tool approval management (GET/POST /api/approvals)
  • Backend tests for machine CRUD — Pairing flow, agent WebSocket, agent manager currently untested

Low Priority

  • Dark/light theme toggle — Allow users to switch between dark and light themes
  • Import/export prompt groups — JSON-based backup and sharing of prompt templates
  • Template ordering — Drag-and-drop reordering of templates within groups
  • Variable validation rules — Define min/max, regex, required/optional per variable
