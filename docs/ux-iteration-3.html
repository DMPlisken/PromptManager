UX Iteration 3 — Machine Card Improvements

Focus: MachineCard component, platform badges, health metrics display

Theme

Iteration 3 focused on the MachineCard component on the Machines page. These cards are the primary way users monitor their fleet of machines, so they need to convey status, health, and platform information at a glance.

Changes Made

Platform Badge Icons

  • Replaced text labels ("macOS", "Windows", "Linux") with emoji icons for instant recognition
  • macOS: apple emoji, Windows: window emoji, Linux: penguin emoji
  • Badge background uses a subtle tinted color matching the platform

Health Metrics Enhancement

  • Added disk space metric (disk_free_gb from health data) alongside CPU and memory
  • Memory now displayed in human-readable format (e.g., "15.2 / 16.0 GB") instead of raw percentage
  • Color-coded bars: green (<70%), yellow (70-90%), red (>90%)

Visual Polish

  • Subtle gradient background using machine color accent (each machine has a user-configurable color)
  • Hover elevation with box-shadow transition (0.2s ease)
  • Null guard on machine_uuid.substring for machines in pairing state (BUG-004 regression prevention)

UX Expert Feedback

Positive: Platform badges with icons are immediately scannable. The gradient backgrounds add personality without overwhelming. Health metrics are now actionable — users can see at a glance which machine is under load.

Suggestions for next iteration:

  • Setup wizard needs UX attention — the 5-step flow could have better progress indicators
  • Connection status dot animation could be smoother
  • Navigation sidebar needs hover states to feel interactive
