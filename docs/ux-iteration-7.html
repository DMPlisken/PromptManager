UX Iteration 7 — Backend: Better Error Messages

Focus: Human-readable error messages in session_manager.py for all failure modes

Theme

Iteration 7 focused on session_manager.py error messages. When sessions fail to create, the error messages shown to users were too technical ("CircuitBreakerOpen", "Connection refused"). This iteration rewrites all error paths to provide actionable, user-friendly messages.

Changes Made

Sidecar Session Errors (Local)

session_manager.py

Before/After comparison of error messages for local sidecar sessions:

Before: "Sidecar is unreachable. Cannot create session."
After: "Sidecar is unreachable (circuit breaker open). Check that the orchestrator sidecar is running on port 9100 and that ORCHESTRATOR_URL is configured correctly in .env."

Added separate handlers for:

  • CircuitBreakerOpen — sidecar has been failing repeatedly
  • ConnectionError — sidecar not running or network issue
  • TimeoutError — sidecar overloaded or unresponsive
  • Generic Exception — unexpected failures with type logging

Agent Session Errors (Remote)

session_manager.py

Improved error messages for remote machine session creation:

Before: "Machine with id 5 not found"
After: "Machine with id 5 not found. It may have been removed. Refresh the machines list and try again."
Before: "Machine 'Office Mac' is offline. Cannot create session."
After: "Machine 'Office Mac' is currently offline. Ensure the agent is running on that machine and has a stable network connection to the server."
Before: "Failed to dispatch session to machine 'Office Mac'"
After: "Failed to dispatch session to machine 'Office Mac'. The agent WebSocket connection may have dropped. Check the agent logs on that machine."

Structured Logging Added

  • Every error path now logs with structlog including session_id, machine details, and error type
  • Log events use descriptive names: session_create_failed_circuit_open, session_agent_machine_offline, etc.
  • Makes it easy to correlate failures in production logs with user-reported issues
