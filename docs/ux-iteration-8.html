UX Iteration 8 — Backend: Agent Manager Logging

Focus: Comprehensive logging for session output messages in agent_manager.py

Theme

Iteration 8 completed the backend polish trilogy (6: cleanup, 7: errors, 8: logging) by adding comprehensive logging to the agent_manager. Every session lifecycle event from remote agents is now logged with structured data for production debugging and audit purposes.

Changes Made

Session Output Logging

agent_manager.py
  • Added logger.debug on every _handle_session_output call with machine_uuid, session_id, sequence number, role, message type, and a 120-char content preview
  • Added warning log when session output arrives without a session ID
  • Debug level keeps production logs clean while allowing verbose tracing when needed
agent_session_output machine_uuid="abc-123" session_id="def-456" sequence=42 role="assistant" msg_type="text" content_preview="I'll start by reading the configuration..."

Session Lifecycle Logging

agent_manager.py
  • _handle_session_started — logs agent_session_started at INFO level with machine_uuid and session_id
  • _handle_session_completed — logs agent_session_completed at INFO level with cost_usd, tokens_in, tokens_out
  • _handle_approval_required — logs agent_approval_required at INFO level with tool_name
  • Pre-existing _handle_session_failed already logged at WARNING — no change needed

Logging Strategy

EventLevelRationale
Session output (each message)DEBUGHigh volume — only enabled when tracing
Session startedINFOImportant lifecycle milestone
Session completedINFOSuccess with cost data
Session failedWARNINGFailure requiring attention
Approval requiredINFOUser action needed
Missing session IDWARNINGProtocol error from agent
